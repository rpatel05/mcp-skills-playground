# Weekly Brief

Weekly summary of product metrics, experiments, and insights for team sharing.

## What it covers

1. **Week-over-week trends** for key metrics
2. **Experiment results** from completed tests
3. **Notable wins** (positive metric movements)
4. **Risks identified** (negative trends or data quality)
5. **Recommendations** for next week

## Output Format

**Week of [Date]**

**🏆 Wins**
- [Metric] improved by X%

**🚩 Risks**
- [Metric] declined by X%

**🧪 Experiments**
- [Experiment]: [Result]

**📊 Key Metrics**
- DAU: X (↑Y%)
- Conversion: X% (↓Y%)

**💡 Recommendations**
1. [Actionable next step]
2. [Actionable next step]

## Best Practices

- Format for easy skimming
- Include specific numbers
- Make recommendations concrete
- Keep to 1 page equivalent
